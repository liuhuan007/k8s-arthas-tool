## 1. 实施概述

### 1.1 实施目标

实现"可用、可审计、可解释"的手动/半自动诊断闭环，包含Agent SDK集成。

### 1.2 核心交付物

| 模块 | P0交付 | 验收标准 |
|------|--------|---------|
| **Skill Registry** | 导入、校验、发布到capabilities | 能够导入Skill并发布 |
| **Workflow Engine** | DSL步骤执行、错误处理、执行记录 | 能够执行Skill的DSL步骤 |
| **Agent Tool Gateway** | 受控工具暴露、权限控制、审计 | Agent只能调用白名单工具 |
| **Agent SDK集成** | 多SDK适配、自动降级、会话持久化 | 支持CodeBuddy/Fallback切换 |
| **诊断能力** | 能力卡片、参数表单、轮询执行 | 用户可以选择能力并执行 |
| **执行记录** | task_logs + step_logs + audit_logs | 完整的执行日志记录 |

### 1.3 技术栈

| 层级 | 技术 |
|------|------|
| **后端** | Python + Flask + SQLite |
| **前端** | 原生JavaScript + CSS |
| **Agent SDK** | CodeBuddy Agent SDK + FallbackAgent |
| **数据库** | SQLite（WAL模式） |
| **实时通信** | HTTP轮询（P0），WebSocket（P2） |

## 14. 风险管理

### 14.1 风险清单

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| Agent SDK生产环境不稳定 | 高 | 中 | FallbackAgent降级方案 |
| SQLite并发写入瓶颈 | 中 | 低 | WAL模式+异步写入 |
| 前端多标签页状态不同步 | 中 | 中 | BroadcastChannel+sessionStorage |
| 自定义Skill安全漏洞 | 高 | 低 | 发布前校验+审计 |
| 工期延误 | 中 | 中 | 预留缓冲时间 |

### 14.2 应急预案

| 场景 | 应急预案 |
|------|---------|
| Agent SDK不可用 | 切换到FallbackAgent |
| SQLite性能不足 | 考虑迁移到PostgreSQL |
| 前端组件库缺失 | 优先实现核心组件 |
| 工期延误 | 调整P0范围，推迟P1功能 |
