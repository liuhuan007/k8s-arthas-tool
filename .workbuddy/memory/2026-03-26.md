## 采样日志跨连接污染问题（2026-03-26）
- **问题**：切换 Arthas 连接后，旧连接的采样日志仍然显示在界面上
- **根因**：旧连接的轮询定时器没有被正确停止，继续调用 `pfLog` 输出日志到 DOM
- **解决方案**：
  1. 新增 `_pfPollingForConn` 变量，记录当前轮询是为哪个连接服务
  2. `pfPoll()` 函数只在 `_pfPollingForConn === _currentConnId` 时才输出日志到 DOM
  3. 所有启动轮询的地方设置 `_pfPollingForConn = _currentConnId`
  4. 任务完成或切换连接时清空 `_pfPollingForConn`
- **效果**：每个连接的采样日志互不干扰，切换后只显示当前连接的日志

## 采样任务状态保存时机错误修复（2026-03-26）
- **问题**：切换连接后，其他连接的采集按钮显示"⏳ 采集中"且无法点击
- **根因**：保存任务状态时，代码在 `clearInterval(_pfPollTimer)` **之后**才检查 `_pfPollTimer`，此时 `_pfPollTimer` 已为 `null`，导致任务状态永远不会被保存
- **修复**：
  - 在停止轮询**之前**先检查并保存任务状态（用 `hadRunningTask` 变量）
  - 添加 `hasRunningTask` 变量使逻辑更清晰
- **效果**：切换到没有任务的连接时，按钮正确显示"▶ 开始"且可点击

## 采样日志轮询 UI 更新串台修复（2026-03-26）
- **问题**：切换连接后，旧连接的轮询仍更新新连接的进度条和标签，造成视觉串台
- **根因**：`pfPoll()` 函数中进度条、标签、按钮等 DOM 更新代码没有检查 `_pfPollingForConn === _currentConnId`
- **修复**：将所有 DOM 更新（日志输出、进度条、标签、按钮、Toast）都放入 `if (_pfPollingForConn === _currentConnId)` 判断内部
- **效果**：只有当前连接的轮询才会更新 UI，彻底解决视觉串台问题

## 采样任务状态保存到错误连接的键（2026-03-26）
- **问题**：切换连接后，新连接的按钮显示"采集中"，日志面板显示旧连接的日志
- **根本原因**：保存任务状态时，代码在 `_currentConnId = connId` **之后**才保存状态，此时 `_currentConnId` 已经是新连接的 ID，导致把旧连接的状态保存到了新连接的键下
- **修复**：
  - 在设置 `_currentConnId` 之前，先用 `oldConnId` 保存旧的连接 ID
  - 保存任务状态到 `oldConnId`，而不是 `_currentConnId`
  - 切换到没有任务的连接时，清空日志面板显示"等待启动..."
- **效果**：彻底解决切换连接后的状态混乱问题
