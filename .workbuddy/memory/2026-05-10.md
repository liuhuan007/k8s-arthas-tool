### Arthas 连接无限循环修复（Critical Bug）

**根因**：`ensure_agent_running()` 在 HTTP 可达时，通过 `_find_arthas_pids()` 查找 `arthas-boot` 进程。但 Arthas attach 到目标 JVM 后，`arthas-boot` 进程正常退出，agent 在 JVM 内运行。找不到 PID 就判定为 zombie port → 返回 `REINSTALL_NEEDED` → `connect_arthas()` 递归重试（无上限）→ 无限循环。

**修复**：
- `arthas_agent.py`：HTTP 可达 = agent 在运行，找不到 arthas-boot PID 是正常的，不再返回 REINSTALL_NEEDED
- `connection.py`：`connect_arthas()` 增加 `_reinstall_retries` 上限（最多 1 次），超限返回明确错误
- `pod_apis.py` / `server.py`：移除外层重复的 REINSTALL_NEEDED 重试

### 连接中心步骤状态同步修复

**问题**：刷新页面后恢复 Arthas 连接时，步骤 2/3 按钮不更新。

**根因**：`_restoreActiveConnection()` 只设 `window._connState`，但 `updateConnectionButton()` 读局部 `_connState`，且 `ConnectionStore` 未更新，订阅回调不触发。

**修复**：三个恢复路径全部同步：
- 局部 `_connState` + `window._connState`
- `ConnectionStore.setState()` 触发订阅回调
- `ConnectionStore.setCurrentConnection()` + `updateConnection()`
- `_podConnId` — `upgradeToArthas()` 依赖此变量，恢复时必须设置
- **显式调用** `updateConnectionButton()` + `updateFeatureTabs()` — ConnectionStore 订阅回调不保证同步触发按钮更新
