# 2026-03-27 工作记录

## Python 版本升级

### 问题
- 服务器 Python 3.6 不支持现代语法
- 项目代码使用了 Python 3.9+ 和 3.7+ 的特性

### 解决方案
1. **安装 Miniconda**：在服务器上安装 Miniconda
2. **创建 Python 3.10 环境**：`conda create -n arthas python=3.10`
3. **激活环境**：`conda activate arthas`

### 代码现代化更新

#### 类型注解（Python 3.9+）
- ❌ `from typing import List, Dict, Tuple`
- ✅ 直接使用内置泛型：`list[str]`, `dict[str, Any]`, `tuple[int, str, str]`

#### subprocess 调用（Python 3.7+）
- ❌ `subprocess.run(cmd, stdout=PIPE, stderr=PIPE, universal_newlines=True)`
- ✅ `subprocess.run(cmd, capture_output=True, text=True)`

#### 修改文件
- `server.py`：简化 typing imports，使用内置泛型
- `profiler_backend.py`：恢复 capture_output=True，更新类型注解
- `pod_monitor.py`：简化 typing imports，更新类型注解

### 部署状态
- ✅ 语法验证通过（Python 3.13）
- 📦 打包文件：`python310-upgrade.tar.gz`
- ⏳ 等待服务器 SSH 恢复后部署

### 服务器命令
```bash
# 初始化 conda
/root/miniconda3/bin/conda init bash
source ~/.bashrc

# 创建环境
conda create -n arthas python=3.10 -y
conda activate arthas

# 部署
cd /root/k8s-arthas-tool
tar xzf python310-upgrade.tar.gz
pip install -r requirements.txt
python server.py
```
